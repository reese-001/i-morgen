    Overview

    • For time and memory complexity, we measure the complexity of the function with respect to the input
    • This analysis is known as asymptotic analysis, which means we analyze the time a function takes to complete as the input tends towards infinity

    Example

    • Assume a = [] and consider three different functions operating on list a
      • f1(a) => 1 + a[0]
      • f2(a) => sum(a)
      • f3(a) => pair(a)
    • If len(a) == 1
      • All three functions will complete basically instantly
    • If len(a) == 10000
      • f1(a) will still complete basically instantly
      • f2(a) will still complete basically instantly
      • f3(a) may take a few seconds
    • If len(a) == 100000000
      • f1(a) will still complete basically instantly
      • f2(a) may take a few seconds
      • f3(a) would not complete
    • How can we summarize this?
      • It does not make sense to assign time values since it is dependent on the size of the varibale
      • We can say however that f2 and f3 are sensitive to the size of the array whereas f1 is not
      • The measure of complexity is not just a measure of speed, but a measure of speed with respect to the size of the input increases
      • f1 = O(1)
        • Known as constant time. The speed of f1 is independent of the size of the input
        • Elementary operation
      • f2 = O(N)
        • Known as linear time. As the size of the input grows, the time to complete the function scales linearly
      • f3 = O(N^2)
        • Known as exponential time. For each number in the array, we retraverse the entire array
