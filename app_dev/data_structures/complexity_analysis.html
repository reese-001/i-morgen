    • Overview

    • Why measure complexity?
      • Allows us to determine the scalability of algorithms and data structures, and to make trade-offs between time and space efficiency
      • Helps to design efficient algorithms and data structures that can handle large amounts of data and solve problems in an acceptable amount of time and memory
    • Complexity can be measured in (at least) four ways
      • Time
        • This measures the amount of time it takes for a data structure to complete its operations
        • Expressed in Big O notation
      • Space
        • This measures the amount of memory required by a data structure
        • Expressed in Big O notation
      • Best/Worst/Average-case analysis
        • This type of analysis measures the complexity of a data structure under different conditions, such as when the input data is sorted, unsorted, or partially sorted
      • Amortized analysis
        • This type of analysis provides an average estimate of the time complexity of a sequence of operations on a data structure, taking into account any overhead incurred by periodic operations such as resizing an array
