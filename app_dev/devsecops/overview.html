    • Overview

    • DevSecOps is a practice that integrates security considerations into the software development life cycle (SDLC) and the operations of an organization
    • It aims to bridge the gap between development, security, and operations teams, by promoting collaboration, communication, and automation of security activities throughout the entire SDLC
    • Expands upon the principles of Devops by shifting security considerations and testing to the left with the goal of identifying vulnerabilities early in the SDLC cycle - and prior to deployment
    • Key focus areas
      • Automating Security
        • Pipeline based security
          • Pre commit hooks
            • Scripts that are executed prior to a GIT commit
            • Used to enforce certain rules or checks on the code being committed
            • In the context of IT application security, pre-commit hooks can be used to
              • Validate code for security vulnerabilities
              • Format code to meet security standards
              • Ensure the code is in compliance
          • Source composition analysis (SCA)
            • Analyzes the components and dependencies of an appplication to identify known vulnerabilities and potential security risks
            • Typically involves scanning source code and metadata to identify included packages and compares them to databases of known vulnerabilities
            • Helps identify issues early in the process
            • Products include
              • Sonatype Nexus
                • Through component analysis, it provides insights into the security, quality, and licensing of components
              • Black Duck Hub
                • Provides component inventory management, vulnerability management, policy management and pipeline integration
              • Snyk
                • Provides vulnerability management, continuous monitoring, community vulnerability database and pipeline integration
              • Trufflehog
                • Can scan a Git repository and find secrets (e.g., API keys, passwords, etc.) that were accidentally committed to the repository
          • Static App Security Testing (SAST) (Dependency-Check, Requires.IO, Retire.js)
            • Analyzes source code of an application for potential vulnerabilities without executing
            • Looks for patterns and anomalies that indicate vulnerabilities such as
              • Missing input validations
              • Hard coded passwords
              • Buffer overflows
            • Products include
              • SonarQube
              • Checkmarx
              • Veracode Static Analysis
              • Snyk
          • Dynamic Security Testing (DAST)
            • Similar to SAST, except it anaylzes the executing of an application
            • Does this by sending various inputs and observing output
            • Goal is to identify security risks that may not be observable through SAST
              • Cross site scripting (XSS) link
              • Cross site request forgery (CSRF) link
              • SQL Injection attacks link
            • Products include
              • Acunetix
              • AppScan
              • Checkmarx

    Definitions

    • Vulnerability
      • Weakness in software or hardware that can be exploited by attackers to cause harm.
    • Expliot
      • Piece of code or technique that takes advantage of a vulnerability to carry out an attack.
    • Threat
      • Potential danger that can cause harm to an organization or system. A threat can be made up of one or more exploits that target known vulnerabilities.
