Contents

    • Learning Rate

    • Neural networks are typically training by gradient descent
    • We start with random weights in our network and try to reduce the cost function over a number of epochs
    • Learning rate is how far the model can go in trying to reduce the cost function
    • If viewed on a 2D graph, it would be points that ideally would converge on the lowest point
    • Consequences of each approach
      • Smaller LR = higher training time
      • Higher LR = possibility of overshooting the correct solution

    Batch Size

    • Defines how many training samples are used within each batch of each epoch
    • This is relevant to being able to avoid becoming stuck in local minima
      • Smaller batch sizes provide more 'wiggle' and thus are more effective at avoiding becoming trapped in local minima
      • Consequently, larger batch sizes can become stuck and land on the wrong solution
    • Random shuffling at each epoch can make this look inconsistent from run to run
      • Some runs you may get stuck in local minima and other runs you will not
