Introduction to Cloud, AWS and SageMaker

What does a typical machine learning lifecycle look like?

  • AWS documentation
  • Illustration
  • Steps (defined in class)
    • Business Context
      • Define the problem and business value
    • Pre-process
      • Clear, transform and create new features
      • SageMaker offers Feature Store and Data Wrangler
    • Train
      • Model building, training, evaluation
      • SageMaker offers Spot Training and One Click Train
    • Tune
      • Change parameters to enhance performance
      • SageMaker offers Auto Model Tune and Experiments
    • Deploy
      • Publish the model into production for use
      • SageMaker offers Pipelines and One Click Deploy
    • Monitor
      • Manage, Validate, Manage Costs
      • SageMaker offers Clarify and Model Monitor
  • SageMaker offers the following general tools
    • Studio: platform where we implement these services
    • Jumpstart: collection of prebuilt models, algos and frameworks
    • Autopilot: helps with entire lifecycle in a no code way
    • Debugger: validating ML code
