    EC2 On Demand

    • Pay for what you use
      • Linux or Windows: billing per second after the first minute
      • All other OS's: billing per hour
    • Highest cost but no upfront payment
    • No long term commitment
    • Recommended: short term and un-interrupted workloads where you can't predict how the application will behave

    EC2 Reserved Instances

    • Note: more details here
    • Large discount compared to on-demand
    • Reserve a specific instance attributes (instance type, region, tenaency, OS
    • Reservation period: 1 year (+ discount) or 3 years (+++ discount)
    • Payment options: No upfront (+), partial upfront (++), all upfront (+++)
    • Reserved instance scope: regional or zonal
    • Buy and sell in Reserved Instance Marketplace
    • Recommended: steady state (ie database)
    • Convertable Reserved Instance Type
      • Can change the EC2 instance type, instance family, OS, scope, tenancy

    EC2 Savings Plans

    • Get a discount based on long term usage (similar to Reserved Instances)
    • Commit to a certain type of usage (ie. $10/hr for 1 or 3 years)
    • Usage beyond EC2 savings plans is billed at the on demand price
    • Locked into a specific instance family and region (ie: M5 in us-east-1)
    • However you are flexible across:
      • instance size (ie: m5.xlarge, m5.2xlarge)
      • OS (ie: Linux, Windows
      • Tenancy (ie: host, dedicated, default

      Spot Instances

      • Most aggressive discounts
      • You can lose them at any point in time if yor max price is less then the current spot price
      • Useful for:
        • batch jobs
        • Data analysis
        • Image processing
        • Any distributed workload
        • Workloads with a flexible start and end time
      • Not suitable for critical jobs or databases

      EC2 Dedicated Host

      • Physical server with EC2 instance capacity fully dedicated to you
      • Useful for complaince requirements and for server bound licenses (ie: per socket, per core)
      • Purchasing options
        • On demand: pay per second for active dedicated host
        • Reserved: 1 or 3 years (no upfront, partial upfront, full upfront)

      EC2 Dedicated Instances

      • Instances run on hardware thats dedicated to you
      • However you may share the hardware with other instances in the same account
      • No control over instance placement

      EC2 Capacity Reservations

      • Reserve on-demand instances in a specific AZ for any duration
      • You always have access to EC2 capacity when you need it
      • No time commitments (create/cancel any time), no billing discounts - only purpose is to reserve capacity
      • Combine with regional reserved instances and savings plans to benefit from billing discounts
      • You are charged at on-demand rates whether you run instance or not
      • Recommended: short term, un-interrupted workloads that need to be in a specific AZ

      Vacation Analogy

      • On demand: coming and staying in resort whenever we like, we pay full price
      • Reserved: like planning ahead and if we plan to stay a long time, we may get a discount
      • Savings plans: Pay a certain amount per hour for a certain period and stay in any room
      • Spot instances: the hotel allows people to bid for empty rooms and the highest bidder gets it. You may be kicked out any any time
      • Dedicated host: Book an entire building at the resort
      • Capacity reservations: Book a room for a period and pay full price, even if you dont stay in it
