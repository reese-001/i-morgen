Git Checkout

Contents

  • Overview
  • Commands
  • Quizlet (external)

    • Overview

    • A "checkout" allows the following actions
      • Switching between different versions of a target entity
        • Said another way, git checkout alllows you to navigate between the branches created by git branch
        • Checking out a branch
          • Updates the files in the working directory to match the version stored in that branch
          • Tells Git to record all new commits on that branch
      • Restoring files to a specific version
    • Benefits
      • Makes it easy to try new experiements without fear of destroying existing functionality
      • Allows you to work on many unrelated features at the same time
    • Dangers
      • Allows you to overwrite changes in your working directory
      • Be sure you understand what you are doing (and potentially have a backup) before executing

    Commands

    Intent Command
    To switch to a different branch in your repository git checkout <branch-name>
    To revert to specific version git checkout HEAD -- <file name>
    To create a new branch and switch to it at the same time git checkout -b <new-branch-name>
    Checkout a specific commit in your repositor git checkout <commit-hash>
