    create branch, modify code, commit branch and merge to main

    • git checkout -b <new-branch-name>
    • ## modify code
    • git add .
    • git commit -m "pushing new feature code"
    • git push
      • We get an error here because we are no longer on main
      • Use the command that is returned: git push --set-upstream origin new-branch-name
    • Now that the feature branch is checked in, it has to be merged to main
      • Within CodeCommit (similar to GitHub), perform the following steps
        • Select "Pull requests"
        • Ensure the destination and source are correct and confirm the code is mergable
        • Give the pull request a name and select "Create pull request"
        • Review "Changes" tab and if appropriate, select "Merge"
        • Choose the correct merge type (ie FF, Squash and Merge, 3-Way), optionally select "Delete source branch ..." and select "Merge pull request
