Why Bother?

  • Utility computing model
    • If you want electricity in your home, you dont build the power facilities from start
    • Rather, you pay the utility to use what has already been created
  • Access to the global infrastructure
    • Cloud providers have a worldwide presence
    • By leveraging this, you can go global yourself within a short time after creating an account
  • Scalable platform
    • For example, you could create a 1000+ node environment and never worry about where that infrastructure is coming from
  • Opex vs Capex
    • Trade long term expenses for low monthly fees
  • Dynamic capacity provisioning and scalability
    • Allows you to dynamically build a scalable and resilient environment
    • Regardless of the size of your company, if you are on-prem only you will be constrained and overpaying
    • Cloud aggregrates usage to achieve higher economies of scale
  • Availability
    • The cloud platforms will allow monitoring and restoration of impacted componentns
  • Business continuity
    • A natural disaster that hits a specific data center can be mitigated by distributing your work
  • Managed services
    • Allows you to skip to management of things like databases, servers, and other functions
  • Automation
    • You can do anything with code, such as IaC

Brief History of Cloud Computing

  • In 1961, Professor John McCarthy said a "time sharing model will result in a future where computing power will be sold as a utility" - and this is what is happening
  • Around the same time, JCR Licklider from ARPANET conceptualized an "intergalatic network" where computers would be connected to a common network
  • The mainframes on the 1960's gave way to distributed computing in the 1970's
  • In 1991, Tim Berners Lee came up with the idea of a World Wide Web
  • In 1999, virtualization became a reality with VMware
  • Also in 1999, Salesforce.com deployed their CRM software over the web
  • 2006 AWS introduced infrastructure as a service
  • Followed in 2008 with VPC, EBC, ELB, Autoscaling
  • Also in 2008 Google Computer Engine was made available
  • In 2009 Azure came online

Three Types of Cloud

  • Infrastructure as a Service (IaaS)
    • Similar to a large company renting an empty office building and they will furnish it completely
    • Examples include bare metal AWS/GCP/Azure
  • Platform as a Service (PaaS)
    • Similar to smaller company needing a few floors of an existing office with carpet, cubicles and furniture already there
    • Examples include Elastic Beanstalk, Openshift
  • Software as a Service (SaaS)
  • Similar to an individual looking for a "plug and play" type office space - you just being a laptop
  • Examples include Gmail, Dropbox, and anything you can access over the web without installing anything

Cloud Deployment Models

  • Public cloud
    • AWS, GCP, Azure, Digital Ocean and others
    • Anyone can access if they create an account
    • Shared infrastructure
  • Private cloud
    • This is something you build and you own it
    • Why is it a cloud then?
      • It shares many features such as provisioning, automation, and other dynamic features
      • But its available within your own datacenter/parameters
    • Tools like OpenStack help build your own private cloud platform
    • Some public cloud providers are enabling you to create private clouds with their templates
  • Hybrid cloud
    • Companies that use both private and public
    • Example may be that some apps need to scale and will be hosted in the cloud for bursting
    • A virtual private link will connect private and public clouds and even isolated from public access

Major Cloud Providers

  • Primary cloud providers
    • AWS
    • GCP
    • Azure
  • They all offer the following services
    • Compute
      • You will be able to create a unit of compute (container or server)
    • Managed databases
      • Both sql and nosql databases
    • Storage
      • Object storage: stores as objects and can be accessed via api or other process
      • Block storage: similar to disk drives
    • Network services
      • Allows you to set bounds on your applications
      • Can create firewall, load balancer, permissions, etc
    • CI/CD tools
    • User management
