    • What is it?

    • RDS is a fully managed database service that makes it easier to set up, operate, and scale a relational database in the cloud. RDS supports a variety of database engines, including MySQL, MariaDB, PostgreSQL, Microsoft SQL Server, Oracle, and Amazon Aurora.
    • With RDS, you can create and manage a database instance with just a few clicks in the AWS Management Console. RDS automatically patches and backs up your database, and it provides high availability and failover support to ensure your database stays up and running even in the event of a failure.
    • RDS provides a variety of benefits for developers and businesses. It can help you reduce the time and cost of setting up and maintaining a database, and it offers scalability and flexibility to support the needs of your application. You can also use RDS to host multi-tenant applications or to build a data warehouse.
    • Storage backed by EBS (gp2 or io1)
    • You cannot SSH into these instances
    • What databases are support?
      • Postgres
      • MySQL
      • MariaDB
      • Oracle
      • MS SqlServer
      • Auora (AWS proprietary database)

    Storage Auto Scaling

    • Helps you increase storage on your RDS DB instance dynamically
    • When RDS detects you are running out of free database storage, it scales automatically
    • Set a Maxiumum Storage Threshold
    • Automatically modify storage if
      • Free storage is less then 10% of allocated storage
      • Low storage lasts for more then 5 minutes
      • 6 hours have passed since last modification

    RDS Read Replicas for Read Scalability

    • What is it>?
      • In the event of read latency, the RDS database will create up to 5 replicas of itself in the following ways:
        • With AZ
        • Cross AZ
        • Cross region
      • These replicas will be populated with async replication - eventually consistant
      • These replicas can be promoted to their own DB
      • Do not use write ops
    • Use Cases
      • You have a production database that is taking normal load
      • You want to run a reporting application on this db
      • Create a read replica to run the new workload there
    • Network Costs
      • For RDS Read Replicas within the same region, there is no additional fee
      • Cross region replication will incur costs
    • RDS Multi AZ (DR)
      • SYNC replication to the standby RDS instance or the transaction fails
      • Automatic app failover to standby
      • Read Replicas can be setup as a Multi AZ for DR
    • How to go from Single AZ to Multi AZ
      • Zero downtown operation
      • Just click "Modify DB"

    Aurora

    • Background
      • Postgres and MySql are both supported as Aurora DB - meaning your drivers will work as if Aurora was a Postgres or MySql db
      • Cloud optimized for 5x performance over MySql on RDS. 3x performance over Postgres on RDS
      • Storage automatically grows in increments of 10GB, up to 128 TB
      • Can have 15 replicas while MySql has 5. Replication process also faster
      • Failover is instantaneous
      • Costs 20% then other RDS systems
    • High Availability and Read Scaling
      • 6 copies of your data across 3 AZ's
        • Only needs 4 copies out of 6 for writes
        • Three copies out of 6 for reads
        • Self healing with peer-to-peer replication
        • Storage is stripped across 100s of volumes
      • One Aurora Instance takes writes (master)
      • Automated failover from master in less then 30 seconds
      • Master + up to 15 Aurora Read Replicas serve reads

    Authentication

    • Supports IAM authentication on the following three databases
      1. MariaDB
      2. MySql
      3. PostgreSQL
    • With this authentication method, you don't need to use a password when you connect to a DB instance. Instead, you use an authentication token.
