What Is It?

  • AWS CloudWatch is a monitoring and observability service provided by AWS
    • Allows you to collect and track metrics, including custom metrics and metrics from AWS services such as EC2, RDS, and Lambda
    • Allows you to collect and monitor log files from AWS resources, third-party applications, and custom applications, and provides features for analyzing and visualizing log data
    • Provides real-time insights into your AWS environment, including metrics and log data, which can help you identify issues, troubleshoot problems, and optimize performance
  • CloudWatch supports dashboards and visualization tools, which allow you to create custom dashboards and charts to visualize your data and gain insights into your AWS environment
    • Enables you to monitor metrics, alarms, and events in real-time, and can be customized to display the data most relevant to your use case
  • CloudWatch allows you to set alarms and trigger automated actions in response to events or changes in resource usage, which can help you automate operational tasks and ensure that your systems are running smoothly
    • Alarms can be set on individual metrics, such as CPU utilization or network traffic, and can trigger automated actions such as scaling or notifications
    • Integrates with other AWS services, such as EC2 Auto Scaling, to enable automated scaling based on CloudWatch metrics and alarms
  • CloudWatch provides APIs and other tools for automating management tasks, which can help you streamline your operations and reduce the time and effort required to manage your AWS environment

CloudWatch Components

  • There are six different components of CloudWatch
    • Metrics
    • Alarms
    • Dashboards
    • Logs
    • Insights
    • Events

Metrics

  • CloudWatch provides metrics for every service in AWS
  • A metric is a variable to monitor (CPUUtilization, NetworkIn, etc)
  • Metrics belong to namespaces
    • A namespace is a container for CloudWatch metrics, which provides a way to group related metrics together
    • Metrics that share a namespace are typically related to the same resource or application
    • May have common characteristics such as units of measure or dimensions
  • Dimensions is an attribute of a metric (instance id, environment, etc)
    • A dimension is a name-value pair that helps to identify and categorize the data being collected by the metric
    • By specifying dimensions, you can more easily filter and analyze the metrics based on specific attributes
  • You can have up to 30 dimensions per metric
  • Metrics have timestamps
  • Can create CloudWatch dashboards of metrics

Alarms

  • Alarms are used to trigger notifications for any metric
  • Various options exist to create alarms (sampling, %, max, min, etc)
  • Alarms can take one of three states
    • OK
    • INSUFFICIENT_DATA
    • ALARM
  • Period: the length of time in seconds to evaluate the metric
  • There are three types of alarm targets
    • EC2 instances
      • Stop
      • Terminate
      • Reboot
      • Recover
    • ECR
      • Trigger autoscaling action (scale out/in)
    • SNS
      • Send notification to SNS
      • From there you can do almost anything
