Allow Internet Access on VPC: Reference

  1. Create VPC
    • Typical CIDR block is 10.0.0.0/16
    • By default you get a NACL and Route Table
    • DNS hostnames is disabled by default. Turn it on.
  2. Create Internet Gateway
    • Will initially create in detached state
    • Attach it to your VPC
  3. Modify or Create new Route Table
    • If you create a new one, set it as the main route table
    • Edit routes on route table
    • Destination: 0.0.0.0/0
    • Target: IGW
  4. Assuming you are making your own subnets...
    • Select CIDR range smaller then selected from VPC
    • For example if CIDR was 10.0.0.0/16, select 10.0.0.0/24 or 10.0.1.0/24
    • After creation, go back and make it public by editing subnet settings
    • Ideally create at least 3 public subnets and 1 private
  5. Create Route Table Associations
    • Route tables are created by default when you create the subnets
    • However in this example we created a new route table with no data and modified the private subnet to associate to the new route table
