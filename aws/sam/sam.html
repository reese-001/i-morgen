Serverless Application Model (SAM)

What does it do? 

  • Extension of CloudFormation that lets you define serverless applications
  • SAM is both an AWS CLI took and a CloudFormation Macro. Effortless to define and deploy serverless apps
  • When compared to CloudFormation, SAM allows at least a 50% reduction in code
  • CLI makes it easy to run, package and deploy serverless applications or Lambdas

Valid SAM Resource Types 

  • AWS::Serverless::Function
    • This resource type is used to define an AWS Lambda function and its associated configuration, such as the function's runtime, memory size, and execution role.
  • AWS::Serverless::Api
    • This resource type is used to define an Amazon API Gateway REST API and its associated resources, such as stages, methods, and integration types.
  • AWS::Serverless::SimpleTable
    • This resource type is used to define an Amazon DynamoDB table and its associated primary key and optional global secondary index.
  • AWS::Serverless::Application
    • This resource type is used to define an AWS Serverless Application Repository (SAR) application and its associated resources, such as IAM roles and access policies.
  • AWS::Serverless::LayerVersion
    • This resource type is used to define an AWS Lambda layer and its associated version.
  • AWS::Serverless::StateMachine
    • This resource type is used to define an AWS Step Functions state machine and its associated resources, such as states and transitions.
  • AWS::Serverless::LogGroup
    • This resource type is used to define an Amazon CloudWatch Logs log group and its associated retention policy.
  • AWS::Serverless::EventSourceMapping
    • This resource type is used to define an event source mapping for an AWS Lambda function, which specifies the event source and triggers for the function.
  • AWS::Serverless::LayerVersionPermission
    • This resource type is used to define permissions for an AWS Lambda layer version, allowing other AWS accounts or IAM users to use the layer.
  • AWS::Serverless::FunctionEventInvokeConfig
    • This resource type is used to define the event invoke configuration for an AWS Lambda function, which specifies the retry and concurrency settings for the function when it is invoked asynchronously.
