Continuous Integration (CI)

  • What is it?
    • Software development practice that involves regularly merging code changes from multiple developers into a shared repository and then automatically building and testing the code
    • The goal of CI is to catch bugs and other issues as early as possible in the development process, making it easier and less expensive to fix them
    • Keep in mind the security aspect of this. Link
  • How does it work?
    • Code is pushed to a central repository
    • Code is built
    • Code is tested
    • Code is deployed
  • Benefits
    • Automated builds
    • Automated testing
    • Continuous feedback
    • Faster releases
    • Collaberation
  • Tool set
    • Open Source
      • Jenkins
        • Offers a high degree of flexibility and extensibility
        • Can build, test, and deploy software across a variety of platforms, languages, and tools
        • Can be more difficult to set up and configure than some other CI tools, and it may require more maintenance and resources to run
      • Travis CI
        • Similar to Circle CI
        • Cloud-based CI service that offers support for a wide variety of programming languages and platforms
        • Provides a variety of tools and integrations to help you automate your testing and deployment workflows, and it offers a free tier for open source projects
        • Can be more limited in terms of customization options and some users have reported issues with slow build times and unexpected behavior.
      • Circle CI
        • Similar to Travis CI
        • Cloud-based CI service that allows you to build and test your code in a variety of environments
        • Known for its ease of use and fast setup, which can make it a good choice for small to medium-sized projects or teams with limited resources
        • CircleCI can be more expensive than other CI tools, and some users have reported issues with slow build times and limited customization options
    • Commercial
      • TeamCity
        • Product offering from JetBrains
      • Bamboo
        • Product offering from Atlassian and integrates well with Jira and Hipchat

Continuous Delivery/Deployment (CD)

  • What is it?
    • Extension of the Continuous Integration (CI) process that focuses on automating the software release process
    • Enable the rapid and frequent release of high-quality software with minimal manual intervention
    • Code changes are automatically built, tested, and deployed to production, allowing teams to release new features and fixes to customers quickly and with confidence
    • Continuous delivery vs continuous deployment
      • Continuous Delivery
        • Ability to deploy often using automation
        • May involve a manual step to approve prior to production deployment
      • Continuous Deployment
        • Expands upon continuous delivery
        • Full automation straight to production
  • How does it work?
    • Code changes are made and pushed to a repo
    • CI runs
    • Artifacts are created (ie. Docker)
    • Environments are provisioned
    • Artifacts are deployed
    • Automated (or manual) testing and approval
    • Deploy to production
  • Benefits
    • Automated testing
    • Continuous deployment
    • Continuous feedback
    • Faster time to market
    • Improved collaboration
  • Tool set
    • Open Source
      • Jenkins CD
      • Spinnaker
    • Commercial
      • Azure Devops
      • GitLab
