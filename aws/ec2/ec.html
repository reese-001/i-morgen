EC2

What is it?

  • EC2 is a highly configurable server
  • Resizable compute capacity
  • Takes minutes to launch new instances
  • Anything and everything on AWS uses EC2 instance underneath

Instance Types and Usage

  • General Purpose
    • A1, T3, T3a, T2, M5, M5a, M4
    • Balance of compute, memory, and network resources
    • Use case: web and code repos
  • Compute Optimized
    • C5, C5n, C4
    • Ideal for compute bound applicaitons that benefit from high performance processor
    • Use case: scientific modeling, dedicated gaming servers, ad server images
  • Memory optimized
    • R5, R5a, X1e, X1, High Memory, z1d
    • Ideal for fast performance worklaods that process large datasets in memory
    • Use case: in memory caches, in memory databases, real time big data analytics
  • Accelerated Optimized
    • P3, P2, G3, F1
    • hardware accelerators, or co-processors
    • Use case: machine learning, computation finance, seismic analysis, speech recognition
  • Storage optimized
    • I3, I3en, D2, H1
    • Ideal for high sequential read and write access to very large datasets on local storage
    • Use case: NoSql, in memory or transactional databases, data warehousing

Instance Profile

  • Instead of embedding your AWS credentials in your code so your instance has permissions to access certain services, you attach a role to an instance via Instance Profile
  • The instance profile holds reference to the role
  • The EC2 instance is associated with the instance profile
  • Automatically created when launching instance via console

Placement Groups

  • Placement groups allow you to choose the logicial placement of your instances to optimize for ...
    • Communication
    • Performance
    • Durability
  • Cost: Free
  • Types of placement groups
    • Cluster: packs instances close together in AZ. Cluster cannot be multi-AZ
    • Partition: spreads instances across logical partitions. Partitions do not share underlying hardware
    • Spread: each instance is placed in a seperate rack. Must be multi-AZ

UserData

  • A script that will automatically run when launching EC2
  • Used to install package, apply updates or anything else

MetaData

  • From within your EC2 instance, you can access information via http://169.254.169.254/latest/meta-data
  • Examples:
    • /public-ipv4
    • /ami-id
    • /instance-type
  • Can combine metadata with userdata scripts to perform all sorts of advanced aws staging automation

User Data

  • Allows you to specify data (such as a script or configuration file) that should be executed when an EC2 instance is launched.
    • The user data specified when launching an Amazon Elastic Compute Cloud (EC2) instance is only executed once, when the instance is launched
    • Once the user data has been executed, it is not executed again unless the instance is stopped and restarted.
  • The user data is passed to the instance as part of the instance metadata, and can be accessed from within the instance using the EC2 metadata service.
  • You can use the user data feature to automate tasks such as installing software packages, configuring the instance, or joining the instance to a domain
  • The user data that is specified when launching an Amazon Elastic Compute Cloud (EC2) instance is executed with root privileges on the instance

Misc Notes

  • Burst balance: refers to the balance of credits that are available to a burstable performance instance (t2, t3, and t3a instance types).
    • When a burstable performance instance is launched, it is allocated a certain number of credits that can be used to support bursts of higher CPU usage
    • The instance uses these credits to support bursts of CPU usage that exceed the baseline CPU performance of the instance
    • The number of credits that are allocated to an instance is based on the instance type and the size of the instance
    • As the instance uses credits to support bursts of CPU usage, the burst balance of the instance decreases.
    • When the burst balance reaches zero, the instance's CPU performance will be limited to the baseline performance of the instance until the balance is replenished.

Types of Reserved Instances

  • Scheduled Reserve
    • Are a type of EC2 instance that allows you to reserve capacity in advance for a specific time period.
    • Scheduled Reserve instances are ideal for workloads that have predictable start and end times, or that require a consistent number of instances over a fixed period of time.
    • They are available for one or three year terms.
    • Limitations
      • must be launched within the specified time period. If you do not launch the instances within the time period, the reservation will expire and you will not be able to use it.
      • Cannot be used with Auto Scaling or with EC2 Auto Scaling groups.
      • Cannot be stopped or terminated, but they can be modified or unmodified
  • Standard Reserve
    • EC2 instance reservation that allows you to purchase a reservation for a specific instance type in a specific Availability Zone.
    • In general, Scheduled Reserved Instances are more specific and are best suited for workloads that have a predictable schedule, while Standard Reserved Instances are more general and are suitable for any workload that runs on EC2 instances.
  • Convertable Reserve
    • A type of EC2 instance reservation that allows you to purchase a reservation for a specific instance type and then exchange it for a different instance type at a later time
    • This can be useful if your workload requirements change over time and you need to use a different instance type to meet those requirements.

Dedicated Host vs Dedicated Instance

  • Dedicated Hosts
    • AWS Dedicated Hosts are physical servers that are dedicated for your use and that you can use to host one or more Amazon EC2 instances.
    • You can purchase Dedicated Hosts on a monthly basis and specify the number of instances that you want to run on each host
    • Dedicated Hosts give you full visibility and control over the physical infrastructure that your instances are running on, including the hardware, operating system, and virtualization technology.
  • Dedicated Instance
    • AWS Dedicated Instances are Amazon EC2 instances that run on hardware that is dedicated to a single customer
    • hey are launched in a VPC and can be used like any other EC2 instances, with the added benefit of being physically isolated from instances that belong to other customers
    • They are not tied to a specific host and can be migrated between hosts as needed
    • Dedicated Instances are a good option if you want the benefits of running on dedicated infrastructure but do not need the full visibility and control that Dedicated Hosts provide.
