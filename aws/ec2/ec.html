EC2

What is it?

  • EC2 is a highly configurable server
  • Resizable compute capacity
  • Takes minutes to launch new instances
  • Anything and everything on AWS uses EC2 instance underneath

Instance Types and Usage

  • General Purpose
    • A1, T3, T3a, T2, M5, M5a, M4
    • Balance of compute, memory, and network resources
    • Use case: web and code repos
  • Compute Optimized
    • C5, C5n, C4
    • Ideal for compute bound applicaitons that benefit from high performance processor
    • Use case: scientific modeling, dedicated gaming servers, ad server images
  • Memory optimized
    • R5, R5a, X1e, X1, High Memory, z1d
    • Ideal for fast performance worklaods that process large datasets in memory
    • Use case: in memory caches, in memory databases, real time big data analytics
  • Accelerated Optimized
    • P3, P2, G3, F1
    • hardware accelerators, or co-processors
    • Use case: machine learning, computation finance, seismic analysis, speech recognition
  • Storage optimized
    • I3, I3en, D2, H1
    • Ideal for high sequential read and write access to very large datasets on local storage
    • Use case: NoSql, in memory or transactional databases, data warehousing

Instance Profile

  • Instead of embedding your AWS credentials in your code so your instance has permissions to access certain services, you attach a role to an instance via Instance Profile
  • The instance profile holds reference to the role
  • The EC2 instance is associated with the instance profile
  • Automatically created when launching instance via console

Placement Groups

  • Placement groups allow you to choose the logicial placement of your instances to optimize for ...
    • Communication
    • Performance
    • Durability
  • Cost: Free
  • Types of placement groups
    • Cluster: packs instances close together in AZ. Cluster cannot be multi-AZ
    • Partition: spreads instances across logical partitions. Partitions do not share underlying hardware
    • Spread: each instance is placed in a seperate rack. Must be multi-AZ

UserData

  • A script that will automatically run when launching EC2
  • Used to install package, apply updates or anything else

MetaData

  • From within your EC2 instance, you can access information via http://169.254.169.254/latest/meta-data
  • Examples:
    • /public-ipv4
    • /ami-id
    • /instance-type
  • Can combine metadata with userdata scripts to perform all sorts of advanced aws staging automation

Misc Notes

  • Burst balance: refers to the balance of credits that are available to a burstable performance instance (t2, t3, and t3a instance types).
    • When a burstable performance instance is launched, it is allocated a certain number of credits that can be used to support bursts of higher CPU usage
    • The instance uses these credits to support bursts of CPU usage that exceed the baseline CPU performance of the instance
    • The number of credits that are allocated to an instance is based on the instance type and the size of the instance
    • As the instance uses credits to support bursts of CPU usage, the burst balance of the instance decreases.
    • When the burst balance reaches zero, the instance's CPU performance will be limited to the baseline performance of the instance until the balance is replenished.
