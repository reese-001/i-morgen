Security Groups

What are they?

  • Act as a virtual firewall at the EC2 instance level
  • Each SG contains a set of rules that filter traffic coming into and out of EC2 instances
  • All traffic is blocked unless a rule specifically allows it. There is no need for deny rules
  • There is a one to many relationship between SG's and EC2 instances
  • SG's are stateful. If traffic is allowed inbound, it is also allowed outbound
  • Cannot block specific IP's (see NACL)

Use Cases

  • Web site wanting to talk to backend database (scenario 1)
    • You can specify the source to be an IP range or specific address (/32 is a specific address)
    • Illustration
  • Web site wanting to talk to backend database (scenario 2)
    • You can specify the source to be another security group
    • Illustration
  • Instance can belong to multiple security groups
    • Rules are permissive, meaning if you have one SG which has no Allow and another to Allow, then it will Allow
    • Illustration

Limitations

  • You can have up to 10,000 SG's in a region (default is 2,500)
  • You can have 60 inbound rules and 60 outbound rules per group
  • 16 SG's per ENI (default is 5)
