KMS

What does it do? 

  • kms makes it easy to create, control, and rotate encryption keys
  • kms is a multi tenant Hardware Security Module (HSM)
  • kms can be used with CloudTrail to audit access history

Where can you find it? 

  • RDS, CodeCommit, S3, CodeDeploy, Glacier, SNS, SQS, DynamoDB, EC2, Xray, ElasticCache and Codebuild (among many others)

What is a Hardware Security Module (HSM)

  • Specialized hardware for storing your encryption keys
  • Designed to be tamper-proof
  • Stores keys in memory and never to disk
  • NOTE: CloudHSM is a single tenant HSM solution to provide stricter compliance to FIPS 140-2 Level 3
  • KMS adheres to FIPS 140-2 Level 2

Customer Master Keys

  • CMKs are the primary resources in AWS KMS and a logical representation of a master key
  • CMK's contains metadata such as:
    • key id
    • creation date
    • description
    • key state
  • CMK's contain the key material used to 
    • encrypt
    • decrypt data
  • KMS supports symmetric and asymmetric CMK's
    • example of symmetric: AES-256 (S3)
    • example of asymmetric: RSA (EC2 key pairs)

Definitions

  • Encryption: process of encoding a message or information in such a way that only authorized parties can access it
  • Cryptographic Keys: string of data that is used to lock or unlock cryptographic functions, including authentication, authorization and encryption
  • Master key: used to encrypt all other keys on a system
  • Envelope encryption: using a key to encrypt another key
