SSL Basics

Intro

  • SSL cert allows traffic between your clients and your load balancer to be encrypted in flight
  • SSL refers to Secure Socket Layer and is used to encrypt connections
  • TLS refers to Transport Layer Security and is a newer version of SSL
  • TLS certs are primarily used, but people still refer to SSL
  • Public SSL certs are issued by Certificate Authorities (CA)
  • Examples include
    • Comodo
    • Symantec
    • GoDaddy
    • GlobalSign
    • Digicert
    • Letsencrypt
    • and more
  • SSL certs have an expiration date (you set) and must be renewed

Load Balancer

  • Traffic is encrypted between client and load balancer
  • Traffic is not encrypted betweeen load balancer and target - but it is within a private VPC
  • Load balancer uses X.509 certificate (SSL/TLS server cert)
  • You can manage certs using ACM (AWS cert manager)
  • HTTPS listener
    • You must specify a default certificate
    • You can add optional list of certs to support multiple domains
    • Clients can use SNI (Server Name Indication) to specify the hostname they reach
    • Ability to specify a security policy to support older versions of SSL/TLS

Server Name Indication

  • SNI solves the problem of loading multiple SSL certificates into one web server (to serve multiple web sites)
  • Its a newer protocol and requires the client to indicate the hostname of the target server in the initial SSL handshake
  • The server will then find the correct certificate, or return a default one
  • Works with ALB, NLB or Cloudfront
