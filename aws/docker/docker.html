Docker

What is it?

  • Third party tool designed to make it easier to create, deploy and run apps by using containers
  • Compared to VM's ...
    • Containers make better use of space
    • Containers provide better app isolation
    • These contribute to containers avoiding some of the config conflicts, security problems and resource management issues seen in VM's

Dockerfile components

  • Example
  • FROM: use another docker image as a basis for this Dockerfile
  • RUN: execute any commands in a new layer on top of the current image
  • WORKDIR: change the default folder for future commands
  • COPY: copy files or folders from local computer to image
  • ENTRYPOINT: command executed when the container is started. Cannot be overrided
  • EXPOSE: listen on specified ports at runtime
  • CMD: default arguments for ENTRYPOINT

Docker components

  • docker build: build an image from a Dockerfile
  • docker ps: list of containers
  • docker run: run a command in a new container
  • docker exec: run a command in a running container
  • docker images: list of images
  • docker push: push an image to a repo
  • docker pull: pull an image from a repo
