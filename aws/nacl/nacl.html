NACL (Network Access Control List)

What is it?

  • An optional layer of security that acts as a firewall for controlling traffic in and out of subnets
  • NACL's are stateless - meaning incoming rule will not be applied to outgoing traffic
  • VPC's automatically get a default NACL
  • Subnets are associated with NACL's. Subnets can only belong to a single subnet
  • Each NACL contains a set of rules that can allow or deny traffic into or out of subnets
  • Each NACL is assigned a rule number and corrosponds to the order of evaluation (lowest->highest)
    • Highest rule number is 32766
    • Recommended to work in 10 or 100 increments
  • You can allow or deny traffic - even at the single IP level. Note: this is not possible with Security Groups

Use Case

  • In the event of a malicious activity originating from a single IP
  • If you determine you will never need to SSH into an instance, you can add a deny for those subnets
