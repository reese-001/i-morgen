Understanding YAML

Overview

  • YAML stands for "YAML Ain't Markup Language." It is a human-readable data serialization language that is commonly used for configuration files, data storage, and communication between services.
  • AWS CloudFormation is a service that enables you to use programming languages or a simple YAML file to model and provision, in an automated and secure manner, all the resources needed for your applications across all of your AWS accounts.
  • In CloudFormation, a YAML file is used to define a "stack," which is a collection of AWS resources that you want to create or update as a single unit. The YAML file specifies the resources to create, as well as their properties and dependencies.
  • A CloudFormation template is written in YAML or JSON, and it consists of six main sections:
    • Resources: (Required) This section defines the AWS resources that you want to create or update. Each resource has a type (e.g., Amazon S3 bucket, Amazon EC2 instance), a unique identifier, and a set of properties that specify its configuration.
    • Outputs: This section specifies values that you want to output from your CloudFormation stack. You can use these outputs to pass data between stacks or to display information to users.
    • Conditions: This section lets you specify conditions that control whether certain resources or resource properties are created or updated.
    • Mappings: This section lets you create maps of keys and values that you can reference in other parts of your template.
    • Parameters: This section lets you specify values that you want to pass into your CloudFormation stack when you create or update it. These values can be used to customize the resources that are created or updated.
    • Metadata: This section lets you specify additional information about the CloudFormation stack, such as the stack's description or the version of the CloudFormation template.
    • Transform: This is used to specify the name of one or more AWS CloudFormation macros that you want to use in your template. Typically associated to serverless applications
  • You can use the AWS CloudFormation CLI or the AWS Management Console to create and manage stacks based on your YAML templates. You can also use CloudFormation as part of your continuous integration and delivery (CI/CD) pipelines, to automate the deployment of your infrastructure and applications.

Notes

  • Intrinsic Functions in templates are used to assign values to properties that are not available until runtime. They usually start with Fn:: or !. Example: !Sub or Fn::Sub.
