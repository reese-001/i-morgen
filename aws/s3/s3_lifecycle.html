S3 Lifecycle Management

Contents 

  • Overview
  • Lifecycle Rules
  • S3 Analytics

    • Overview

    • Allows us to move files between different storage classes
    • See diagram
    • From any level, you can go down any number of levels
    • For example, if you know your objects will not be used often, migrate them to Standard IA
    • Alternativelly, if you do not need fast access to them, move to Glacier or Glacier Deep Archive
    • Lifecycle Rules allows us to automate this task

    Lifecycle Rules

    • Transition actions: configure objects to transition to another storage class
      • Move objects to Standard IA 60 days after creation
      • Move to Glacier for archiving after 6 months
    • Expiration actions: configure objects to expire after some time
      • Access log files can be set to delete after 365 days
      • Delete old versions of files
      • Delete incomplete multi part uploads
    • Rules can be created for specific prefixes (s3://mybucket/s3/*)
    • Rules can be created for specific object tags

    S3 Analytics

    • Helps decide when to transition objects to the right storage class
    • Recommendations for Standard and Standard IA
      • Does not work for One-Zone IA or Glacier
    • Report updated daily
    • 24-48 hours to see data be generated
    • Good first step to put together lifecycle rules
