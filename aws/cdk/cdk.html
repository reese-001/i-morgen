Cloud Developer Toolkit (CDK)


Contents
  • Overview
  • Install
  • Useful Commands
  • Getting Started

    • Overview

    • Transpiler. Takes input from popular languages (Python, Java) and converts it to CloudFormation, which in turn creates the resources
    • CloudFormation is a declarative language. You tell it what you want, and it figures out how to get it done
    • Compare to CDK which is imperative
    • YouTube (Be a Better Dev)
    • YouTube (freeCodeCamp.org)

    Install

    • Prerequisites (when using Python)
      • NPM
        • Link
      • pip
      • AWS CLI
        • Link
      • CDK
        • npm install -g aws-cdk
        • Validate: sdk version

    Useful Commands

    • For a complete list of commands, execute "cdk --help"
    • cdk list
      • List all stacks in an application
    • cdk synthesize
      • Transpiles your code into CloudFormation
    • cdk bootstrap
      • If the code you create generates a CloudFormation script that is over 50kb, you will need to use this command
      • Additionally, if there are AWS resources that are required in order to deploy your stack you can accomodate this via bootstrap
    • cdk deploy
    • cdk destroy
    • cdk diff

    Getting Started

    • To start out with a bare bones application with nothing but raw files that requires you to fill everything else in
      • cdk init app --language python
    • To start with a framework that can be modified
      • cdk init sample-app --language python
      • This will create a template with the following files
        • source.bat (or source if on Linux/Mac)
          • Script to activate your virtual environment
        • requirements.txt
          • As expected ...
          • It seems this functionality was previously encapsulated in a file called "setup.py"
        • readme.txt
          • Contains a listing of commands that are offered - good reference
        • app.py
          • Entry point for the application
            • Creates an instance of the Core class, which is the construct provided by AWS CDK and serves as the building block for cloud apps
            • The "app" method returns a new instance of the Core class that is used to define cloud resources in the AWS CDK
            • The returned instance can then be used to interact with and manage the defined cloud resources, such as defining AWS Lambdas, Amazon S3 buckets, Amazon DynamoDB tables, and more
          • Creates stacks
            • Defaults to creating the CdkGettingStartedStack and takes the app variable and the stack name as parameters
            • What is a stack?
              • Creates logical groupings of stacks
              • For example, you could create a stack for your infrastructure and another stack for a dashboard
              • Stacks can then be deployed independently since some deploys can be time consuming
        • cdk_getting_started/cdk_getting_started.py
          • The default stack performs the following
            • Creates a queue via the sqs library
            • Creates an sns topic
            • Add subscription between topic and queue
          • However, typically remove this and insert your own code here
      • To get started, install dependencies
        • Activate your venv by executing the source.bat file
        • pip install -r requirements.txt
