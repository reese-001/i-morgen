Lambda

What is it?

  • AWS Lambda is a serverless, event-driven compute service that lets you run code for virtually any type of application or backend service without provisioning or managing servers.
  • You can trigger Lambda from over 200 AWS services and software as a service (SaaS) applications, and only pay for what you use.

Lambda Aliases

  • An alias is a pointer to a specific version of a Lambda function, and you can use aliases to control traffic to different versions of a function
  • You can use an alias to specify the percentage of traffic that should be directed to a particular version of the function, and you can easily update the alias to point to a different version if needed

Lambda Authorizer

  • API Gateway feature that uses a Lambda function to control access to your API.
  • A Lambda authorizer is useful if you want to implement a custom authorization scheme that uses a bearer token authentication strategy such as OAuth or SAML, or that uses request parameters to determine the caller's identity.

Memory and Computing Power

  • Memory is the principle lever available for controlling the performance of a function as is adjustable between 128 MB to 10,240 MB
  • Adding more memory proporationally adds for CPU

How To

  • Deploy a function that makes use of a Python library not otherwise available to Lambda
    • In this case, we will install "requests"
    • pip install --target ./package requests
    • cd package
    • zip -r9 ${OLDPWD}/function.zip .
    • cd ..
    • zip -g function.zip lambda_function.py
    • aws lambda update-function-code --function-name my-function --zip-file fileb://function.zip
