Code Commit


Contents
  • What is it?

  • Authorization
  • Setup

What is it?

  • A version control service hosted by Amazon Web Services that you can use to privately store and manage assets (such as documents, source code, and binary files) in the cloud
  • It is a fully managed source control service that makes it easy for companies to host secure and highly scalable private Git repositories
  • Based on Git and similar to GitHub, BitBucket and GitLab

Features

  • Complies with many compliance programs (ie HIPAA)
  • Repos encrypted at rest as well as in transit
  • Can handle repos with large number of files or branches, large file sizes and lengthy revision histories
  • No limit on the size of your repo or on the file types you can store
  • Keeps your repos close to your other production resources in AWS
  • Integrates with IAM to control which users have access to specific repos

Authorization

  • IAM user SSH keys
    • These are SSH keys that are associated with an IAM user and can be used to authenticate the user when accessing CodeCommit repositories over SSH.
  • IAM user HTTPS credentials
    • These are a username and password combination that are associated with an IAM user and can be used to authenticate the user when accessing CodeCommit repositories over HTTPS.
  • AWS CodeCommit credential helper
    • This is a command-line tool that allows you to use Git credentials stored in the AWS Secrets Manager to access CodeCommit repositories. The credential helper allows you to use Git with CodeCommit without having to store your credentials in a file on your local machine.
  • Git credentials
    • These are a username and password combination that can be used to authenticate with CodeCommit repositories over HTTPS. Git credentials are typically stored in a file on your local machine and are used by the Git command-line tool to authenticate with CodeCommit.
  • SSH keys
    • These are SSH keys that are associated with a user's local machine and can be used to authenticate the user when accessing CodeCommit repositories over SSH
  • ** Note that IAM user SSH keys and IAM user HTTPS credentials are specific to IAM users, while Git credentials and SSH keys are not tied to any particular AWS identity.

Setup

  • To create the repo, you must initially provide a repo name. Optionally you can provide descriptions and tags
  • There are three ways of connecting to the repo
    • HTTPS
      • For non root users, credentials are available via IAM user/security credentials
    • SSH (non root access)
      • For non root users, credentials are available via IAM user/security credentials
    • HTTPS (GRC)
      • GRC = git-remote-codecommit
      • Provides a simple method for pushing and pulling code from CodeCommit repositories by extending Git
      • It is the recommended method for supporting connections made with federated access, identity providers, and temporary credentials

CodeCommit vs Git

  • For more information on git, see link
  • Hosting
    • Git: Can be hosted in many locations
    • CodeCommit: Fully managed service hosted by AWS
  • Integration with other AWS services
    • Git: This can generally happen, but it requires additional setup
    • CodeCommit: Tightly integrated into CodeBuild and CodePipeline
  • Access control
    • Git: Offers fine grained controls, but the specifics will vary depending on the platform and configuration
    • CodeCommit: Also offers fine grained controls but controlled via IAM
  • Scalability
    • Git: Can scale as needed, but you need to consider infrastructure requirements
    • CodeCommit: Designed to support large teams and complex workflows
  • Cost
    • Git: Generally free, but may accure costs associated with hosting and managing the infrastructure
    • CodeCommit: Pay as you go model, costs based on number of repos and users - as well as data transfer and storage usage
