What is it?

  • A version control service hosted by Amazon Web Services that you can use to privately store and manage assets (such as documents, source code, and binary files) in the cloud
  • It is a fully managed source control service that makes it easy for companies to host secure and highly scalable private Git repositories
  • Based on Git and similar to GitHub, BitBucket and GitLab

Features

  • Complies with many compliance programs (ie HIPAA)
  • Repos encrypted at rest as well as in transit
  • Can handle repos with large number of files or branches, large file sizes and lengthy revision histories
  • No limit on the size of your repo or on the file types you can store
  • Keeps your repos close to your other production resources in AWS
  • Integrates with IAM to control which users have access to specific repos

Authorization

  • IAM user SSH keys
    • These are SSH keys that are associated with an IAM user and can be used to authenticate the user when accessing CodeCommit repositories over SSH.
  • IAM user HTTPS credentials
    • These are a username and password combination that are associated with an IAM user and can be used to authenticate the user when accessing CodeCommit repositories over HTTPS.
  • AWS CodeCommit credential helper
    • This is a command-line tool that allows you to use Git credentials stored in the AWS Secrets Manager to access CodeCommit repositories. The credential helper allows you to use Git with CodeCommit without having to store your credentials in a file on your local machine.
  • Git credentials
    • hese are a username and password combination that can be used to authenticate with CodeCommit repositories over HTTPS. Git credentials are typically stored in a file on your local machine and are used by the Git command-line tool to authenticate with CodeCommit.
  • SSH keys
    • These are SSH keys that are associated with a user's local machine and can be used to authenticate the user when accessing CodeCommit repositories over SSH
  • ** Note that IAM user SSH keys and IAM user HTTPS credentials are specific to IAM users, while Git credentials and SSH keys are not tied to any particular AWS identity.
