API Gateway

What is it?

  • Amazon API Gateway is a fully managed service that makes it easy to create, publish, maintain, monitor, and secure APIs at any scale.
  • With API Gateway, you can create REST and WebSocket APIs that enable your backend services to be accessed by web and mobile applications.
  • API Gateway handles all of the tasks involved in accepting and processing API requests, including traffic management, authorization and access control, monitoring, and API version management.
  • You can use API Gateway to create APIs that access AWS or other web services, as well as data stored in the AWS Cloud. API Gateway also integrates with other AWS services, such as AWS Lambda, Amazon SNS, Amazon Kinesis, and Amazon DynamoDB, making it easy to build serverless architectures and microservices.

Authentication

  • Amazon Cognito User Pools
  • Amazon Cognito Identity Pools
  • IAM
  • OpenID Connect
  • OAuth 2.0
  • Custom authorizers
  • Security Token Service
    • STS is a web service that enables you to request temporary, limited-privilege credentials for AWS Identity and Access Management (IAM) users or for users that you authenticate (federated users).
    • These temporary credentials consist of an access key ID, a secret access key, and a security token.
    • Typically, you use STS if you want to make a one-time call to AWS using the security credentials of an IAM user or a federated user.
    • You can also use STS to get access to temporary security credentials and use them to access a resource that supports STS authentication in a more convenient way than embedding the long-term credentials in your application.
    • For example, you might want to make periodic calls to Amazon EC2 to automatically start or stop your instances based on your predefined schedule.
    • With STS, you can create a script that calls the STS API to get temporary security credentials and then use those credentials to call the Amazon EC2 API.
    • For more information, see Temporary Security Credentials.

    Mapping Templates

    • Mapping Templates allow you to specify how the data returned by the Lambda function should be transformed before it is returned to the client
    • You can use Mapping Templates to convert the data into a different format (such as XML or JSON), to filter out unwanted data, or to modify the data in any way you like.

    Customer Interceptor

    • A piece of code that runs before or after the execution of an AWS Lambda function
    • A custom interceptor can be used to perform additional tasks or to modify the input or output of the Lambda function
    • Custom interceptors are typically implemented as Lambda functions themselves, and they are invoked by the AWS Lambda runtime when a Lambda function is executed.
    • Can be useful for a variety of purposes, such as logging, monitoring, security, or error handling
