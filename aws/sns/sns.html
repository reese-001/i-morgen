SNS

  • Message publishing and processing service (pubsub)
  • Allows fanout to millions of consumers (ie email, http endpoints, sqs, texting)
  • Fully managed and durable with automatic scaling
    • durable = message is guaranteed not be lost (note: this does not imply each subscriber will get the message)
      • uses cross availability zone message storage
    • managed = no need for infra settings 
    • can send to valid AWS endpoints such as SQS and Lambda functions
  • Security
    • can set topic policies to restrict who can publish and subscribe to a topic
    • data encrypted at rest and in transit
  • Two topics to understand
    • topics
    • subscriptions
  • Can be app -> person or app -> app
    • app -> person: text
    • app -> app: asynch communication between systems
