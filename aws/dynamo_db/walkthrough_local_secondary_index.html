Senario: Table is defined as shown. How to query Forum Name and Subject

Option 1

  1. Scan entire table
  2. Apply filter condition on scan where forum == 'xx' and subject = 'yy
  3. Con: expensive since you are performing entire table scan
>

Option 2

  1. Filter for forum name = 'xx'
  2. Scan result set for subject = 'yy'
  3. Con: Still more expensive then required since scanning multiple items

Option 3

  1. Apply a local secondary index on Subject attribute
  2. Query where forumname = 'xx' and define a range key expression where the subject = 'yy'
